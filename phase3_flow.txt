        - 오른쪽 사이드바: AI 차단 권고 패널 확인
        - nav 우측에 로그아웃 버튼 + "보안 대시보드" 타이틀 확인

[v] D-3. 이벤트 통계 차트
        - 차트 영역에 Chart.js 막대 차트 렌더링 확인
          (데이터 없으면 빈 차트 표시, 에러 아님)
        - 10초 후 차트 자동 갱신 확인 (Network 탭에서 /api/dashboard/stats 요청 반복 확인)

[x] D-4. 실시간 이벤트 피드 (SSE)
        - Network 탭 → EventStream 또는 /api/security/stream 연결 확인
        - 별도 창에서 consumer 계정으로 로그인 실패를 3회 시도
        - 대시보드 피드에 LOGIN_FAIL 이벤트 자동 추가 확인
        - 형식: [HH:MM:SS] •dot• LOGIN_FAIL  {ip}  →  {메시지}

[ ] D-5. 이벤트 필터
        - "SQLI" 필터 버튼 클릭 → SQLI 외 항목 숨김 / active 클래스 이동 확인
        - "브루트포스" 클릭 → brute 타입만 표시 확인
        - "전체" 클릭 → 모든 항목 다시 표시 확인

[ ] D-6. AI 차단 권고 패널 (미처리 권고 없을 때)
        - "미처리 권고 없음" 텍스트 표시 확인 (#alertsEmpty 보임)

[ ] D-7. 즉시 AI 분석
        - "즉시 분석 실행" 버튼 클릭
        - 버튼 비활성화 + "분석 중..." 텍스트로 변경 확인
        - 완료 후 버튼 원상 복구 확인
        - AI 권고 카드 생성 시 카드 렌더링 + "미처리 권고 없음" 숨김 확인
          (AI 분석 결과가 없으면 빈 상태 유지, 에러 아님)

[ ] D-8. AI 권고 카드 — 무시
        - 카드가 있는 경우 "무시" 버튼 클릭
        - 해당 카드 DOM에서 제거 확인
        - 카드 전부 제거 시 "미처리 권고 없음" 다시 표시 확인

[ ] D-9. AI 권고 카드 — 차단
        - 카드가 있는 경우 "차단" 버튼 클릭
        - POST /api/admin/users/:id/block 요청 확인 (Network 탭)
        - 성공 시 카드 제거 + 성공 알림 확인
        - 차단된 계정으로 로그인 시도 → 로그인 실패 확인

완료 기준
================================================================
위 체크리스트 항목이 모두 [ ] → [x] 로 완료될 때 Phase 3 통과.
미통과 항목은 해당 JS 파일 또는 API 엔드포인트로 돌아가 수정 후 재확인.

실패 시 돌아갈 위치:
  - 리다이렉트 안 됨       → payment.js / dashboard.js 상단 guard 확인
  - locationStatus 미표시  → setLocationStatus() 호출 경로 확인
  - QR 발급 실패           → Network 탭에서 /api/payments/request 응답 확인
  - 결제 에러 코드 불일치  → ERROR_MESSAGES 매핑(payment.js) 확인
  - SSE 미연결             → auth.middleware.js query token fallback 확인
  - 차트 미렌더링          → Chart.js CDN 로드 여부 / barChart 캔버스 존재 확인
  - AI 분석 버튼 무반응    → POST /api/security/ai-analyze 응답 확인 (DISABLE_AI=true 여부)
